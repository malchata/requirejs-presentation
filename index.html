Need Further Abstraction?

Use define() instead of require()

define() vs. require()

  • Both take the same arguments in the same order: dependencies, then code.
  • define() defines code.
  • require() requires code to run.

scripts/jquery-stuff.js

							
define(["lib/jquery", "lib/modernizr"], function($, Modernizr){
	$(function(){
		// Application Logic Here
	});
});
- Create a new file called jquery-stuff.js
- This code is not immediately executed. require() call is needed in app.js

scripts/app.js

							
require(["lib/jquery"], function($){
	$(function(){
		// Application Logic Here
	});
});
- Remove the function closure argument.
- Change the dependency from "lib/jquery" to "lib/jquery-stuff"

shim

- shim is a way to centralize our dependencies.

deps

							
// Configuration Settings
requirejs.config({
	"paths":{
		"jquery": ["//code.jquery.com/jquery-1.11.2.min", "lib/jquery.min"],
		"modernizr": "lib/modernizr.min",
		"jquery-stuff": "lib/jquery-stuff"
	},
	"shim":{
		"jquery-stuff":{
			"deps": "jquery"
		}
	}
});
- Explain that each entry in "shim" is specific to a particular module.
- A good example would be jQuery UI being listed as a dependency to jQuery.
- If a module has multiple dependencies, explain the use of the array syntax.

Don't We Already Define Dependencies in Modules?

- Short answer: Yes we do. We don't have to use deps.
- But, it is a centralized way to ensure that we are loading the modules we need.
- It can reduce the amount of clutter in our dependency arrays in our modules.

Esoteric Config Options

An Incomplete List:

  • bundles: Maps many modules to a single ID.
  • config: Specify options in the module's namespace (e.g., API keys, version numbers, other module-specific settings.)
  • waitSeconds: How long to wait for a module to load before giving up.
  • skipDataMain: Disables the data-main entry point.
  • urlArgs: Query string variables to append to module requests (useful for overriding caching mechanisms.)

Setting Up a Build Profile

build.js

							
({
	"baseUrl": "scripts",
	"paths":{
		"jquery": "lib/jquery.min",
		"modernizr": "lib/modernizr.min",
		"jquery-stuff": "lib/jquery-stuff"
	},
	"name": "app",
	"out": "scripts/app.built.js"
});
- "baseUrl" points to where app.js resides.
- "paths" needs to map to the values in requirejs.config.
- "name" should be the same name as the script in your data-main entry point.
- "out" is the destination output file.

Building on the Command Line

							
> r.js -o build.js
- This command will produce a minified single file.
- r.js can also be used to minify CSS.

Building via Grunt

  1. Modify package.json to include "grunt-contrib-requirejs" under "dependencies".
  2. Set build options under the requirejs task in Gruntfile.js (These options are the same as in the previous build.js.)
  3. Run the requirejs task at the command line on demand, or bind it to the watch task so that it is generated when scripts are modified.
